### （1）题目意思与生僻概念解释

**题目含义：**
在一个原本从小到大排好序的数组中，寻找数值最小的那个元素。但是，这个数组经过了一次“旋转”操作，不再是完美的整体递增状态。

**概念解释：**

  * **有序数组：** 数字按照从小到大的顺序依次排列的集合，例如 `[1, 2, 3, 4, 5, 6]`。
  * **旋转（Rotation）：** 指的是把数组最后面的某几个数字切下来，整体平移拼接到数组的最前面。例如，把 `[1, 2, 3, 4, 5, 6]` 最后的三个数字 `[4, 5, 6]` 切下来放到前面，数组就变成了 `[4, 5, 6, 1, 2, 3]`。
  * **旋转后的特征：** 旋转后的数组会被截断成两段递增的区间（左半段和右半段）。并且，**左半段的所有数字，都严格大于右半段的所有数字**。而整个数组的最小值，恰好就是右半段区间的第一个数字（即发生断裂的那个分界点）。

### （2）解题思路的由来

从代码的判断逻辑 `nums[mid] <= nums[-1]` 可以看出，这种解法的核心思想是**二分查找（Binary Search）**，并通过与数组的最末尾元素进行对比来定位最小值。

1.  **设立基准物：** 数组的最后一个元素 `nums[-1]` 是一个极佳的天然参照物。因为无论怎么旋转，`nums[-1]` 必定属于右侧的那段递增区间（即便没有旋转，整个数组也可以视作一整个右侧区间）。
2.  **找中间点：** 每次在搜索区间中取一个中间位置 `mid`，提取出 `nums[mid]` 的值。
3.  **判断所属区间：**
      * 若 `nums[mid] <= nums[-1]`：说明中间的数字比最末尾的数字小（或相等），这意味着 `mid` 当前踩在**右侧的递增区间**上。既然身处右侧区间，那么真正的最小值要么就是 `nums[mid]` 本身，要么在它的左边。因此，需舍弃右半侧，向左收缩搜索边界。
      * 若 `nums[mid] > nums[-1]`：说明中间的数字比最末尾的数字大，这意味着 `mid` 当前踩在**左侧的递增区间**上。既然身处左侧区间，这里面的数字全部大于右侧区间的数字，最小值绝对不可能在这里。因此，需舍弃包含当前点在内的左半侧，向右收缩搜索边界。
4.  **逼近结果：** 通过不断将搜索范围砍半，当左右边界重合且相互交错时，所指向的位置即为右半段的起点，也就是全局的最小值所在位置。

### （3）代码框与逐行注释

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        left, right = 0, len(nums) - 1   # 定义搜索区间的左右边界，初始化为整个数组的首尾索引
        
        while left <= right:             # 当左边界小于等于右边界时，说明区间内还有元素尚未排查，继续循环
            mid = (left + right) // 2    # 计算当前搜索区间的正中间位置索引（向下取整）
            
            if nums[mid] <= nums[-1]:    # 判断中间位置的值是否小于或等于数组的最后一个值
                right = mid - 1          # 若条件成立，说明最小值在当前位置或更左侧，故将右边界收缩至中间位置的左边
            else:                        
                left = mid + 1           # 若条件不成立，说明当前处于左侧较大区间，最小值必定在右侧，故将左边界收缩至中间位置的右边
                
        return nums[left]                # 循环结束时，left 指针刚好停留在整个数组最小值的位置，返回该位置的数值
```

### （4）每一行代码的详细中文解释

  * `class Solution:`
    定义一个名为 `Solution` 的类，通常用于封装算法题的标准化解法结构。
  * `def findMin(self, nums: List[int]) -> int:`
    定义一个名为 `findMin` 的函数方法，接收一个名为 `nums` 的整数列表作为输入参数，并声明该函数最终会返回一个整数（即求得的最小值）。
  * `left, right = 0, len(nums) - 1`
    初始化二分查找的双指针。`left` 设为 `0`（数组第一个元素的索引），`right` 设为 `len(nums) - 1`（数组最后一个元素的索引），确立初始的全局搜索闭区间。
  * `while left <= right:`
    开启一个循环结构，循环的维持条件是左指针不超过右指针。只要满足此条件，就代表左右指针之间（包含指针所指位置）还有尚未排查的候选数字。
  * `mid = (left + right) // 2`
    通过将左右边界的索引相加并使用 `//` 进行向下取整除以 2，计算出当前搜索区间正中间的索引值，赋给变量 `mid`。
  * `if nums[mid] <= nums[-1]:`
    核心逻辑判断分支。提取数组在 `mid` 索引处的数值，并将其与数组最后一个位置 `nums[-1]` 的数值进行比较，判断是否小于或等于末尾值。
  * `right = mid - 1`
    当上述判断为真时执行此行。此时说明中间点落在整体较小的右半段区间，最小值必定在当前点或者当前点的左侧。因此，将搜索区间的右边界移动到 `mid` 的前一个位置，缩小排查范围。
  * `else:`
    当 `if` 判断为假（即 `nums[mid] > nums[-1]`）时，执行此分支。
  * `left = mid + 1`
    此时说明中间点落在整体较大的左半段区间，最小值绝对不可能在此处或其左侧，必定在右侧更深处。因此，将搜索区间的左边界移动到 `mid` 的后一个位置。
  * `return nums[left]`
    当 `while` 循环因 `left > right` 而终止时，`left` 指针会精准地落在使条件发生翻转的那个临界点上（也就是右侧区间的第一个元素）。此行代码提取并返回该位置的数值，即为最终答案。
