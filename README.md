### （1）用白话文解释题目意思及生僻概念

**题目白话文解释：**
有一个原本从小到大排好序的数字列表。这个列表被人从中切了一刀，把前面的一段放到了最后面。现在的任务是，在这个被切过、挪过位置的数字列表中，找到那个最小的数字。
此外，题目有一个硬性要求：寻找的过程必须非常快，不能一个一个去挨个看（不能使用普通的遍历），必须使用一种能每次排除一半选项的算法（也就是二分查找法）。

**生僻概念解释：**

  * **升序排列**：数字从小到大依次排列，比如 `[1, 2, 3, 4, 5]`。
  * **旋转**：把数组末尾的元素一个个挪到数组开头。就像是一条珍珠项链，解开扣子后，把右边的一小段挪到左边重新扣上。原本排好序的数组经过旋转后，会变成两段各自升序的子序列。比如 `[1, 2, 3, 4, 5]` 旋转两下变成 `[4, 5, 1, 2, 3]`。前半段 `[4, 5]` 和后半段 `[1, 2, 3]` 各自仍然是升序的。
  * **时间复杂度为 $O(\log n)$**：这是衡量算法运行速度的一个标准。意思是如果数据量翻倍，算法只需要多找一次就能得到结果。在数组中寻找目标，能达到这个速度要求的通常只有“二分查找”算法。

### （2）代码解题思路的来源（基于代码反推）

从提供的这段代码来看，解题思路采用了\*\*“开区间二分查找”\*\*。

1.  **寻找参照物**：在一个旋转过的数组中找最小值，需要一个参照物来判断当前所在的位置。代码选择了数组的最后一个元素 `nums[-1]` 作为参照物。
2.  **两段区间的特性**：由于数组原来是升序的，旋转后分成了左右两段升序序列。
      * 左半段的所有元素都会 **严格大于** 数组的最后一个元素（假设发生了实质性的旋转）。
      * 右半段的所有元素都会 **小于或等于** 数组的最后一个元素。
      * 最小值恰好就是右半段的第一个元素。
3.  **区间设计（开区间）**：代码初始化 `left = -1`，`right = len(nums) - 1`。这代表寻找的范围在 `(-1, n-1)` 之间，两边都是开区间（不包含边界本身）。之所以 `right` 不设为 `n`，是因为 `n-1` 位置的元素被当作了对比的基准，最小值要么在 `n-1` 左边，要么就是 `n-1` 本身。
4.  **二分逼近**：
      * 取中间位置 `mid`。
      * 如果 `nums[mid] < nums[-1]`，说明 `mid` 当前位于“右半段”中，最小值在 `mid` 的左侧，或者 `mid` 本身就是最小值。因此，可以把右边界缩紧到 `mid`（即 `right = mid`）。
      * 如果 `nums[mid] >= nums[-1]`，说明 `mid` 当前位于“左半段”中，最小值一定在 `mid` 的右侧。因此，可以把左边界推到 `mid`（即 `left = mid`）。
5.  **循环结束**：当 `left + 1 == right` 时，开区间 `(left, right)` 内不再有任何整数，循环结束。此时 `right` 所指的位置就是右半段的起点，也就是整个数组的最小值。

### （3）每一行代码的注释

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        left, right = -1, len(nums) - 1  # 初始化二分查找的左右开区间边界
        while left + 1 < right:          # 当区间内还有未检查的元素时继续循环
            mid = (left + right) // 2    # 计算当前区间的中间位置索引
            if nums[mid] < nums[-1]:     # 如果中间值小于数组最后一个值，说明mid在右侧递增区间
                right = mid              # 最小值在mid左侧或就是mid，将右边界缩小至mid
            else:                        # 如果中间值大于或等于数组最后一个值，说明mid在左侧递增区间
                left = mid               # 最小值一定在mid右侧，将左边界扩大至mid
        return nums[right]               # 循环结束时，right指向的就是最小值，将其返回
```

### （4）每一行代码的详细中文解释

  * `class Solution:`
      * 定义一个名为 `Solution` 的类，这是 LeetCode 提交代码的标准格式，用来封装解题方法。
  * `def findMin(self, nums: List[int]) -> int:`
      * 定义一个名为 `findMin` 的函数，接收一个整数列表 `nums` 作为参数，并且指定函数最后会返回一个整数（即找到的最小值）。
  * `left, right = -1, len(nums) - 1`
      * 定义两个指针，分别代表搜索范围的左边界和右边界。这里使用的是开区间概念。`left` 初始指向第一个元素的前一个位置（-1），`right` 初始指向最后一个元素的位置。因为把最后一个元素作为判断基准，所以右边界初始设在这里。
  * `while left + 1 < right:`
      * 这是一个循环条件。意思是只要 `left` 和 `right` 之间至少还有一个数字（即开区间不为空），就继续折半查找。当 `left` 和 `right` 相邻时，查找结束。
  * `mid = (left + right) // 2`
      * 计算 `left` 和 `right` 之间的中间位置索引。`//` 表示整除，结果只保留整数部分。
  * `if nums[mid] < nums[-1]:`
      * 核心判断逻辑。比较中间位置的数字和数组最后一个数字的大小。
  * `right = mid`
      * 如果上面的判断成立（中间数字比最后一个数字小），说明中间数字属于后半段的升序部分。最小值要么是这个中间数字，要么在这个数字的左边。因此，把右侧边界收缩到 `mid`。
  * `else:`
      * 如果上面的判断不成立（中间数字大于或等于最后一个数字）。
  * `left = mid`
      * 说明中间数字属于前半段的升序部分。真正的最小值一定在这个中间数字的右边。因此，把左侧边界推进到 `mid`。
  * `return nums[right]`
      * 当 `while` 循环结束时，`left` 和 `right` 紧挨着。由于逻辑设计上最小值始终被框在 `right` 及 `right` 的左侧，此时 `right` 所指的位置就是整个数组的最小值所在的索引。返回该索引对应的数字即可。
